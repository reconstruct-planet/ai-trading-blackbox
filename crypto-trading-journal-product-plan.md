# 코인 매매일지 웹사이트 제품 계획

작성일: 2026-05-27

## 1. 제품 한 줄 정의

한국어권 개인 코인 트레이더가 자신의 반복 손실 패턴, 감정 매매, 규칙 위반을 데이터로 확인하고 다음 거래에서 같은 실수를 줄이도록 돕는 행동 교정형 매매일지.

## 2. 핵심 포지셔닝

단순 기록장이 아니라 "내가 왜 잃는지 알려주는 개인 트레이딩 블랙박스"로 포지셔닝한다.

기존 글로벌 매매일지 도구들은 이미 자동 임포트, 리포트, AI 코칭, 차트, 스크린샷, 백테스트를 제공한다. 따라서 "다방면 기록"만으로는 경쟁력이 약하다. 이 제품은 한국어, 코인, 선물/레버리지, 감정 매매, 거래소별 비용과 복기 루틴에 집중한다.

## 3. 첫 타깃 사용자

1차 타깃:
- Binance, Bybit, Upbit, Bithumb 중 1개 이상을 쓰는 한국어권 개인 트레이더
- 현물보다 선물/단타/스윙 거래 빈도가 높은 사용자
- 손실 원인을 알고 싶지만 스프레드시트 기록은 오래 못 하는 사용자
- "내가 또 같은 실수를 했다"는 자각이 있는 사용자

초기에는 장기 투자자, 포트폴리오 관리형 투자자, 세무 리포트가 주목적인 사용자는 타깃에서 제외한다.

## 4. 해결할 진짜 문제

사용자는 거래를 기록하지 못해서만 실패하지 않는다. 더 큰 문제는 거래 당시의 판단, 감정, 규칙 위반, 시장 맥락을 나중에 복원하지 못한다는 점이다.

핵심 문제:
- 거래 후 복기를 미루다가 기억이 왜곡된다.
- 손실이 반복되지만 패턴을 숫자로 보지 못한다.
- 손절 미준수, 물타기, 과도한 레버리지, 추격 매수 같은 실수를 감정적으로 합리화한다.
- 거래소별 데이터, 스크린샷, 메모, 시장 상황이 흩어져 있다.
- 기존 저널은 기능이 많아도 입력 부담이 높아 루틴화되지 않는다.

## 5. MVP 원칙

MVP의 목표는 "기록이 가능하다"가 아니라 "일주일 안에 사용자가 자신의 가장 비싼 실수 1개를 발견한다"이다.

MVP는 다음 세 가지 경험을 반드시 제공해야 한다.

1. 빠른 기록: 거래를 30초 안에 저장하거나 CSV/API로 가져온다.
2. 손실 패턴 발견: 가장 돈을 많이 잃게 만든 태그, 시간대, 종목, 레버리지, 감정 상태를 보여준다.
3. 주간 복기: 이번 주 가장 비싼 실수와 다음 주 지켜야 할 규칙을 자동으로 정리한다.

## 6. MVP 범위

### P0 필수 기능

- 이메일/소셜 로그인
- 거래 수동 입력
- CSV 업로드
- Binance 또는 Bybit 중 1개 read-only API 연동
- 포지션 단위 거래 정리
- 현물/선물 구분
- 진입가, 청산가, 수량, 수수료, 레버리지, 손익, 수익률 기록
- 손절가/익절가/계획 R 대비 실제 R 기록
- 전략 태그
- 감정 태그
- 실수 태그
- 거래별 메모
- 거래별 스크린샷 첨부
- 일별 PnL 캘린더
- 주간 리포트: 가장 비싼 실수 3개, 가장 수익성 좋은 조건 3개, 다음 주 체크리스트
- read-only API 키 보안 안내와 키 삭제 기능

### P1 확장 기능

- Upbit/Bithumb CSV 또는 API 연동
- Bybit/Binance 중 MVP에 빠진 거래소 추가
- MFE/MAE 분석
- 시간대별 성과
- 요일별 성과
- 코인/페어별 성과
- 규칙 체크리스트
- AI 주간 복기 요약
- 거래 전 체크리스트
- 모바일 최적화

### 명시적으로 제외

- 자동 매매
- 매수/매도 시그널 제공
- 세무 신고 리포트
- 카피트레이딩
- 소셜 랭킹
- 모든 거래소 동시 지원
- 고급 백테스팅
- 실시간 알림 기반 매매 추천

## 7. 핵심 화면

### 대시보드

첫 화면은 총수익보다 행동 패턴을 먼저 보여준다.

우선순위:
1. 이번 주 PnL
2. 이번 주 가장 비싼 실수
3. 규칙 준수율
4. 감정 태그별 손익
5. 최근 거래

### 거래 상세

거래 상세는 "이 거래를 왜 했고, 왜 이 결과가 났는가"를 복원해야 한다.

필수 블록:
- 거래 요약
- 진입/청산 정보
- 계획 대비 실제 결과
- 태그
- 메모
- 스크린샷
- 동일 패턴 과거 거래
- 다음에 반복하지 않을 규칙

### 주간 복기

사용자가 매주 일요일 또는 월요일에 보는 화면이다.

필수 질문:
- 이번 주 가장 많이 잃은 패턴은 무엇인가?
- 수익 거래와 손실 거래의 가장 큰 차이는 무엇인가?
- 지켰을 때 돈을 번 규칙은 무엇인가?
- 어겼을 때 가장 비싼 규칙은 무엇인가?
- 다음 주에 하나만 고친다면 무엇인가?

## 8. 데이터 모델 초안

핵심 엔티티:

- User

- Trade

- Position
- Tag
- TradeTag

중요 설계 결정:

- API/CSV 원본 데이터는 가능하면 원본 형태로 보존한다.
- 사용자가 수정한 복기 데이터와 거래소 원본 데이터를 구분한다.
- AI 분석은 별도 테이블에 저장하고 원본 거래 데이터를 덮어쓰지 않는다.

## 9. AI 기능 가드레일

AI는 투자 조언자가 아니라 복기 도우미로 제한한다.

허용:
- "이번 주 손실 거래의 공통점은 손절 미준수와 높은 레버리지입니다."
- "비슷한 태그의 과거 거래 12개 중 8개가 손실로 끝났습니다."
- "다음 주 체크리스트에 진입 전 손절가 입력을 추가하는 것을 권장합니다."

금지:
- "지금 BTC를 매수하세요."
- "내일 ETH가 오를 가능성이 높습니다."
- "이 전략은 수익을 보장합니다."
- 특정 코인 가격 예측 또는 시그널 제공

모든 AI 화면에는 "투자 조언이 아니라 개인 거래 기록 분석"이라는 맥락을 명확히 둔다.

## 10. 보안 원칙

API 키는 제품 신뢰의 핵심이다.

필수:
- 출금 권한 없는 read-only 키만 허용
- API 키 입력 전 거래소별 권한 설정 안내
- 저장 시 암호화
- 키 삭제 버튼 상시 노출
- 연결 실패/권한 과다/동기화 실패 상태를 명확히 표시
- 민감 데이터 접근 로그

출금 권한이 있는 키가 감지되거나 사용자가 권한 설정을 확인하지 않은 경우 연결을 막는다.

## 11. 성공 지표

초기 제품의 성공은 가입자 수보다 복기 루틴으로 판단한다.

핵심 지표:
- 첫 거래 기록까지 걸린 시간
- 첫 주간 복기 생성률
- 2주차 재방문율
- 거래당 평균 태그 입력률
- 주간 리포트 열람률
- 사용자가 직접 저장한 "다음 주 규칙" 수
- API/CSV 연동 성공률
- 동기화 실패율

피해야 할 허영 지표:
- 단순 페이지뷰
- 누적 거래 수만 보기
- AI 호출 수
- 대시보드 위젯 수

## 12. 가격 전략

초기에는 무료 + 유료 구독 구조가 적합하다.

무료:
- 수동 입력
- 월 거래 50개
- 기본 캘린더
- 기본 태그

유료:
- 거래소 연동
- 무제한 거래
- 주간 리포트
- 고급 패턴 분석
- AI 복기
- 스크린샷 저장

가격 가설:
- 개인 트레이더용 월 9,900원 또는 14,900원
- 진지한 선물 트레이더용 월 29,000원

초기에는 가격 최적화보다 유료 전환 이유가 되는 "이번 주 가장 비싼 실수" 리포트의 체감 가치를 검증한다.

- CSV 업로드
- Binance 또는 Bybit read-only API 연동

### 4주차: 패턴 분석

- 태그별 손익

- AI 요약 초안

## 14. 주요 리스크

### CRITICAL GAP: 차별화 부족

단순 매매일지로 만들면 기존 제품 대비 약하다. 반드시 "행동 교정"과 "코인 특화"가 중심이어야 한다.

### CRITICAL GAP: API 키 보안 불신

트레이더는 거래소 키 입력을 극도로 꺼린다. read-only, 암호화, 삭제, 권한 검증, 보안 안내가 MVP부터 필요하다.

### WARNING: 입력 부담

감정, 전략, 실수, 메모를 모두 요구하면 사용자는 이탈한다. 기본값, 빠른 태그, 사후 편집, 자동 임포트가 필요하다.

### WARNING: AI 책임 범위

AI가 투자 조언처럼 보이면 법적/신뢰 리스크가 생긴다. 가격 예측과 매매 추천은 제품 범위 밖으로 고정한다.

### WARNING: 거래소별 데이터 정규화

거래소마다 체결, 수수료, funding fee, realized PnL 표현이 다르다. 처음부터 모든 거래소를 지원하지 않는다.
