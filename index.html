THE REAL PROBLEM

기록 부족보다 더 위험한 건, 손실 원인을 기억하지 못하는 것입니다.

트레이더는 거래 직후엔 이유를 안다고 생각하지만, 며칠 뒤 복기할 때는 감정, 진입 근거, 손절 변경, 시장 맥락이 흐려집니다. Blackbox는 거래의 숫자와 행동 기록을 묶어 반복 손실을 드러냅니다.

01 손절가를 낮추고 버틴 거래
02 급등 캔들 추격 진입
03 손실 만회용 레버리지 확대
04 복기 없이 반복되는 같은 실수

PATTERN ENGINE

AI가 잡아야 할 것은 수익률이 아니라 행동 패턴입니다.

01

손실 패턴

종목, 시간대, 전략, 레버리지, 태그를 묶어 가장 비싼 반복 실수를 계산합니다.

02

감정 패턴

FOMO, Revenge, Fear, Calm 상태별 손익 차이를 보여주고 위험한 조합을 감지합니다.

03

규칙 위반

진입 전 손절가, 계획 R, 포지션 크기 같은 약속을 실제 거래 결과와 대조합니다.

04

주간 복기

이번 주 가장 비싼 실수와 다음 주 지켜야 할 단 하나의 규칙을 정리합니다.

COMMAND FLOW

거래소 연결부터 다음 거래 전 체크리스트까지 하나의 루틴으로.

01 Connect

Binance 또는 Bybit read-only API와 CSV를 가져옵니다.

02 Normalize

체결 데이터를 포지션 단위로 묶고 수수료, 레버리지, R 값을 정리합니다.

03 Detect

AI가 반복 손실, 감정 태그, 규칙 위반, 위험한 시간대를 스캔합니다.

04 Review

주간 리포트와 다음 주 체크리스트로 행동을 교정합니다.

SECURITY POSTURE

트레이딩 제품의 신뢰는 API 키 화면에서 결정됩니다.

  • 출금 권한 없는 read-only 키만 허용
  • 권한 과다 감지 시 연결 차단
  • 저장 시 암호화, 언제든 키 삭제
  • AI는 투자 조언이 아닌 거래 기록 분석으로 제한
